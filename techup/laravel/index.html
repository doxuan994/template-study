Installation


Step 1: Install composer

Fast Intructions From My Website

                            Copy $ curl -sS https://getcomposer.org/installer | php
                            Paste to Terminal -> Enter
To See More Offical Details

Go to: https://getcomposer.org/ In the home page of getcomposer click to Browse Packages

Step 2: Rename composer

Fast Intructions From My Website

Move composer.phar folder to bin and Rename composer.phar to composer


                            Copy [sudo] mv composer.phar /usr/local/bin/composer
                            Paste to Terminal -> Enter

Now just run composer in order to run Composer instead of php composer.phar.

To See More Offical Details

Go to: https://getcomposer.org/doc/00-intro.md#globally

Step 3: Install Laravel 5.4

Fast Intructions From My Website

- First, download the Laravel installer using Composer:


                            $ composer global require "laravel/installer"

A message will display after successfully installation:


                            Changed current directory to /Users/[yourMacUsername]/.composer

- Setting up path:


                            $ export PATH="~/.composer/vendor/bin:$PATH"

- Open this file by using terminal:


                            $ nano ~/.bashrc

In the file opened on terminal copy and paste the following:


                            alias laravel='~/.composer/vendor/bin/laravel'
                            source ~/.bashrc
                            laravel

Note: If you want to save the changes you've made, press Control+ O. To exit nano, type Control+ X. If you ask nano to exit from a modified file, it will ask you if you want to save it. Just press N in case you don't, or Y in case you do. It will then ask you for a filename. Just type it in and press Enter.

- Create a directory named blog containing a fresh Laravel installation with all of Laravel's dependencies already installed:


                            $ laravel new blog
To See More Offical Details

Go to: https://laravel.com/docs/5.4

Laravel Tips and Tricks


Addition features that are removed from Laravel 5+

Go to: https://laravelcollective.com/

To start server 


                            $ php artisan serve

To check Laravel version


                            $ php artisan --version
                            $ php artisan -V
MVC Laravel Useful commands:

To generate model and migration (automate creating a database table without doing it manually)


                        $ php artisan make:model [name of model with a capital letter] -m

Note: Shortcut for --migration -m

To create a new migration


                        $ php artisan make:migration [nameofyourmigration]

                        Example: $ php artisan make:migration add_slug_to_users

To upload data to database


                        $ php artisan migrate

To back up your migration when errors occured


                        $ php artisan migrate:rollback
                        $ php artisan migrate:refresh
                        $ php artisan migrate:reset

After deleting migrations file manually, receive "ErrorException: failed to open...

[ErrorException]...failed to open stream: No such file or directory

To delete almost all steps. Okay can be renew again with [$ php artisan migrate]


                        $ php artisan migrate:rollback

To roll back with specified number of steps


                        $ php artisan migrate:rollback --step=[number-of-step]

Model and Migration can be created separetely from each other by the followong command lines:


                        $ php artisan make:model [Name-Of-Your-Model with capital letter]
                    

                        $ php artisan make:migration [name-of-table] —-create=[name-of-model]

Add new column to exiting table:

$ php artisan make:migration [name-of-your-existing-migration] -—table=[name-of-your-table]

Note: If there is an existing database, then do not need to add migration, only need to create model. Migration is used to generate or modify database. Model is needed, way to talk to database.


To create a new controller with all codeblocks created for you

$ php artisan make:controller [name of controller with capital first letters] --resource

Note: If you want to create codeblocks youself do not need to include --resource in your command line above

To list all the routes


                            $ php artisan route:list
